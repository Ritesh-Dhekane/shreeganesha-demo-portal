# Production Deployments & Release References

This directory tracks the progression of approved features from the demo portal into the production storefront environment.

## Purpose
* Document which demo versions or features have been promoted to the production application.
* Keep a log of production release notes, branch tags, and deployment dates.
* Record configuration values or API keys used in production (excluding sensitive secrets).

## File Naming Convention
* Use format: `YYYY-MM-DD-release-vX.X.X.md` (e.g. `2026-07-01-release-v1.0.0.md`).
